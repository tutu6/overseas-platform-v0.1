## 二、全局设计原则

贯穿所有流程和模块的横向准则。

### 原则 1:最小可行性原则(MVP 第一原则)

- 不擅自扩展需求
- V1.0 所有增量功能全部不做
- 所有"锦上添花"的能力都让位于"主链路打通"

### 原则 2:AI 能力"留占位 + 可降级"⭐

**核心立场**:除了明确的 4 个 Agent,业务流程中还有大量节点"可能"用 AI 辅助(如资质 OCR、入驻 AI 初审意见、报价合理性提示等)。这类 AI 能力**优先级靠后**,但**必须留占位**。

**MVP 实施要求**:

| 维度 | 要求 |
|---|---|
| **模型选择** | 用国内平价大模型(成本可控,如 DeepSeek、Qwen 等),具体到节点时再定 |
| **实现程度** | 做"基础"水平即可,不追求精度 |
| **占位强制要求** | 即便暂时不真实接入,**业务逻辑、数据结构、UI 展示位置必须预留** |
| **Mock 标识** | 所有 AI 相关接口 Response 中加 `mock_ai: true` 字段(具体字段名以 API 规范为准) |
| **优先级** | 所有大模型接入排在底座建设和基础功能之后 |

**典型 AI 占位场景(初步识别,后续逐节点确认)**:

| 流程节点 | 潜在 AI 占位点 | 来源 |
|---|---|---|
| 流程 1 供应商入驻 | 资质 OCR(营业执照等) | 已明确 |
| 流程 1 供应商入驻 | 入驻材料 AI 初审意见(给 ADMIN 审核员参考) | 已明确 |
| 流程 2 商品上架 | 国别准入资质 OCR | 与流程 1 OCR 同源 |
| 流程 2 商品上架 | 商品上架 AI 初审意见 | 类比流程 1 |
| 流程 4 询价/报价 | 报价比价(报价比价 Agent) | 已在 4 个 Agent 内 |
| 流程 4 询价/报价 | 报价合理性 AI 提示 | 待定(Q14) |
| 流程 5 订单履约 | 单据 OCR | 待定(Q14) |
| 流程 5 订单履约 | 异常节点 AI 提示 | 待定(Q14) |

**Mock 实现方案**(倾向 B 方案,待确认):

- A 方案:固定文案
- **B 方案**(倾向):基于业务数据模板化生成,嵌入文件名/供应商名等真实数据
- C 方案:直接接入便宜模型做基础版

**与"4 个 Agent"的关系**:

- 4 个 Agent 是**有独立用户入口的 AI 产品**(独立页面/功能按钮)
- 本原则覆盖的是**业务流程内嵌的 AI 辅助**(无独立入口,内嵌在流程节点中)
- 两者技术栈可共用,但产品形态不同

## 三、角色定义(MVP 阶段)

MVP 阶段仅 3 类角色:

| 角色 | 别名 | 说明 |
|---|---|---|
| **供应商 (SUPPLIER)** | — | 海外材料供货方 |
| **采购方 (BUYER)** | **业主** | 两者是同一概念,央企项目方 |

## 四、关键架构决策

### 决策 1:保留 Organization(组织)的设计抽象 ⭐

虽然 MVP 阶段只接入 **1 个业主方**,但数据模型层面必须保留 `Organization` 这一层:

- **采购方用户** 必须挂在某个 Organization 下
- **数据隔离** 以 Organization 为边界(项目、采购清单、询价单、订单都归属某个 Org)
- **权限分配** 未来基于 Organization 做扩展
- 供应商和管理员**暂不**进入这套组织体系(避免过度设计)

> 这是个**关键架构决策**,影响数据库 schema 设计,落实 schema 时第一个就要确认这一点。

### 决策 2:会员/会费机制

- 供应商通过审核后,有一个**加入会员**的环节
- 缴会费是平台主要盈利来源之一
- 只有会员状态的供应商才能上架商品

## 五、MVP 业务流程

### 流程 1:供应商入驻(含会员缴费)

```
供应商注册账号
   ↓
入驻向导:填基本信息 + 上传资质 + 选择主营品类(商品三级分类)
   ↓ [AI 占位:资质 OCR]
提交审核
   ↓ [AI 占位:入驻材料 AI 初审意见,给 ADMIN 审核员参考]
ADMIN 审核
   ├─ 驳回 → 供应商补材料/改信息 → 重新提交
   └─ 通过
        ↓
   缴纳会员费(成为会员)
        ↓
   获得"可上架商品"权限
```

📌 **待提供资料**:**中建三局商品三级分类**

---

### 流程 2:商品上架(含国别准入审批)

```
会员供应商在工作台 → 新增 SKU
   ↓
填规格 + 阶梯报价 + 选择目标出口国别
   ↓
上传国别准入相关资质(按目标国别要求的认证)
   ↓ [AI 占位:国别准入资质 OCR]
提交审核
   ↓ [AI 占位:商品上架 AI 初审意见]
ADMIN 审核(商品信息 + 国别准入资质)
   ├─ 驳回 → 修改/补资质 → 重新提交
   └─ 通过 → SKU 上架商城
```

---

### 流程 3:采购方创建项目与采购清单

```
采购方登录(隶属于某 Organization)
   ↓
创建项目(项目信息:名称 / 目的国别 / 工程类型 / 预算 / 时间窗等)
   ↓
基于项目创建 1~N 个采购清单(一个项目可对应多份采购清单)
   ↓
浏览商城(按品类/国别/搜索)
   ↓
将符合条件的商品加入指定采购清单
```

---

### 流程 4:询价(RFQ) → 报价 → 达成

```
采购方基于采购清单 → 发起询价单
   ↓
询价单审批(流程内审批)
   ↓
审批通过 → 询价单分发给供应商
   ↓
供应商收到询价单 → 进行报价 → 提交报价
   ↓ [AI 占位:报价合理性提示,待定]
采购方查看报价 → 接受 / 驳回
   ↓ [报价比价 Agent:多家比价]
(可能多轮往返)
   ↓
最终达成一致 → 询价/报价流程结束
   ↓
基于最终报价生成订单(合同)主体
```

---

### 流程 5:订单履约(12 节点追踪)

```
订单生成
   ↓
12 节点全链路追踪:
节点1  订单立项
节点2  国别准入审查
节点3  三方合同签约
节点4  工厂排产
节点5  样品确认
节点6  大货生产
节点7  出厂检验
节点8  出运订舱
节点9  海运在途
节点10 目的港清关
节点11 到场验收
节点12 回款结算
   ↓ [AI 占位:单据 OCR / 异常节点提示,待定]
每个节点由对应责任方打卡 + 上传凭证
   ↓
异常状态汇入 ADMIN 风控驾驶舱
```

---

### 流程关系总览

```
[流程1 供应商入驻+会员]
         ↓
[流程2 商品上架]            ← ADMIN 持续维护"国别准入数据库"
         ↓
   商城有可售 SKU
         ↓
[流程3 项目+采购清单]       ← 采购方入口
         ↓
[流程4 询价 ⇄ 报价(可能多轮)]
         ↓
[流程5 订单生成 + 12节点履约]
         ↓
   ADMIN 风控驾驶舱(横向汇聚监控)
```

## 六、AI 智能体范围(MVP 锁定 4 个)

与研发可行性报告 §10.2.3 W10 一致,MVP 阶段交付 4 个 Agent:

| # | Agent | 集成位置 |
|---|---|---|
| 1 | **标准问答 Agent** | 独立页面 + 国别准入页面侧边助手 |
| 2 | **证书审查 Agent** | 供应商入驻审核 / 商品上架国别准入审批流程内嵌 + 独立页面 |
| 3 | **报价比价 Agent** | 询价单详情页"生成比价报告"功能 |
| 4 | **多语种翻译 Agent** | 独立页面(中↔阿/法/英 等技术文档/合同/认证翻译) |

> V1.0 的 5 个 Agent(供应商尽调 / 履约预警 / 风控资料包 / RFQ 自动响应 / 需求挖掘)**全部不做**。
> 流程内嵌的 AI 辅助见**二、全局设计原则 · 原则 2**。

## 七、风控驾驶舱(纳入 MVP,优先级最低)⚠️

### 基本定位

- **不是一条主流程,是横向监控视图**
- **不发起业务**,被动消费流程 1~5 产生的数据
- **MVP 开发优先级最低**,排在 MVP 末尾(等流程 1~5 主链路稳定后再做)

### MVP 锁定 3 个子模块

对应研发可行性报告 §1.3 底线:

| # | 子模块 | 数据来源 |
|---|---|---|
| 1 | **马甲关系图谱** | 供应商工商穿透数据(公司原有自研算法) |
| 2 | **价格异常监测** | 历史报价数据库 + 当前 RFQ 报价 |
| 3 | **合规雷达** | 订单/供应商/资金流数据(规则引擎基础版) |

### 砍掉的部分(不进 MVP)

研发可行性报告 §7.1 一共列了 6 个风控模块,以下 3 个**不做**:

- ❌ 供应商实时风险预警(依赖企查查/天眼查 API 外部对接)
- ❌ 海外项目交付看板(依赖船舶 API/物流 API)
- ❌ AI 智能洞察(LLM 自动生成日报/周报)

### 与流程的数据流向

```
[流程1 供应商入驻]   ──→ 喂数据 → [马甲关系图谱]、[合规雷达]
[流程2 商品上架]     ──→ 喂数据 → [合规雷达]
[流程4 询价/报价]    ──→ 喂数据 → [价格异常监测]
[流程5 订单+12节点]  ──→ 喂数据 → [合规雷达]
                                      ↓
                              ADMIN 风控驾驶舱(实时监控)
```

### 可见性方案(待定)

可见性方案(ADMIN / 采购方 / 供应商分别看到什么)**后续讨论**,见待定点 Q11~Q13。
