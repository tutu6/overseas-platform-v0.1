## 一、背景

信用评估四维度(工商基础 / 资质认证 / 财务健康 / 司法舆情)评分前,需要先拿到每维度的**原始数据**落入 `credit_company_*_data` 四张快照表,再由评分引擎计算。柬埔寨(KH)是首批接入的国别。

原始数据获取有三条候选路径:
1. **Tavily + LLM**:公开网络搜索 + 大模型结构化抽取
2. **自建爬虫**:直接抓取目标站点(MOC 官网 / 媒体站)
3. **商业 API / 用户上传**:付费数据源 / 供应商自报

关键约束:
- **合规**:央企出海项目,直连别国官方站爬取存在法律 / ToS 风险
- **数据稀缺**:柬埔寨公开数据少,官方站反爬、私企信息不公开
- **成本**:MVP 阶段配额 / Token 成本需可控
- **数据形态**:部分维度是结构化字段(工商/财务),部分是非结构化文本(司法舆情)

本 ADR 基于"双源对照 PoC"(`crawler-poc-comparison/`)的实测数据决策。

## 二、决策

**柬埔寨四维度统一以 Tavily + LLM 为主路径获取原始数据;不自建爬虫;以"用户上传(+ 未来 OCR)"补全关键缺失字段;商业 API 列为后续增强。**

- **主路径**:Tavily 搜索 + qwen-plus 结构化抽取(Δ7 已实现:反幻觉、源字段追溯、域名白名单、白名单优先+全网兜底两阶段)
- **不做**:自建爬虫(直连 MOC 官网 / 媒体站)
- **补充**:供应商注册时上传资质证书 / 财报,补结构化字段(Δ9+);商业工商 API(OpenCorporates 等,Δ8+)
- **降级**:任一维度抓不到 → `data_source='missing'`,评分走缺失降级档,详情接口 `evaluation_status` 标识

## 三、选型理由(基于 PoC 实测)

双源对照 PoC 实测(Kampot Cement / KHALIBRE / Acleda Bank,工商+司法两维度,两路对照):

- **自建爬虫**:MOC 官网 + phnompenhpost + khmertimeskh,共 6 次调用**全部 HTTP 403**(MOC 反爬/WAF;两媒体站 Cloudflare JS challenge)
- **Tavily + LLM**:两维度均能产出(工商填充 1~3/7、司法部分召回),稳定可调

| 因素 | Tavily + LLM | 自建爬虫 |
|---|---|---|
| 柬埔寨可达性 | ✅ Tavily 作为搜索中间层已处理目标站反爬/渲染 | ❌ 三站全 403,拿不到任何内容 |
| 合规 | ✅ 不直连目标站,经搜索中间层 | ⚠️ 直连官方站爬取,法律/ToS 风险 |
| 非结构化数据适配 | ✅ 司法舆情等非结构化文本可由 LLM 归纳出字段 | ❌ 爬虫只能取文本,非结构化源无法出字段 |
| 维护成本 | ✅ 无 DOM 依赖,站点改版不影响 | ❌ 反爬对抗 + 各站 DOM 适配 + CF 规则升级 |
| 与 Δ7 / ADR-0003 一致 | ✅ 同一 Tavily+LLM 栈 | — |

## 四、各维度数据源策略(柬埔寨)

> 核心判断:**爬虫负责"取数据",大模型负责"出字段"。源是结构化的,爬虫可直接出字段;源是非结构化的(新闻/网页正文),必须 LLM 归纳。**

| 维度 | 字段性质 | 柬埔寨主路径 | 是否需 LLM 归纳 | 补充手段 | 现状 |
|---|---|---|---|---|---|
| 工商基础 | 结构化(注册号/成立日期/法人…) | Tavily+LLM | 走网页/Tavily → 需 | 用户上传营业执照、商业 API(Δ8) | Tavily 填充 1~3/7,偏低 |
| 资质认证 | 半结构(是否持证) | Tavily+LLM 判布尔 | 需(判断"该公司是否持证") | 用户上传证书 + OCR(Δ9) | 依赖召回质量 |
| 财务健康 | 结构化(营收/负债率/现金流) | Tavily+LLM(上市可得) | 上市财报可结构化 | **用户上传财报**(私企唯一来源) | 柬埔寨多私企 → 大面积缺失 |
| 司法舆情 | 非结构(负面等级/诉讼/失信) | Tavily+LLM | **必须**(无结构化司法库,只能从新闻归纳) | — | 柬埔寨无公开司法库 + 媒体反爬,**最依赖 LLM** |

**逐维度要点**:

- **工商基础**:字段本是结构化,理论上 MOC 官网可直接爬出、无需 LLM;但 PoC 实测 MOC 403 反爬,退到 Tavily+LLM(网页文本 → LLM 归纳)。
- **资质认证**:ISO/ISC 目录结构化,但"某公司是否持有某证"需从网页判断归属,依赖 LLM 或人工。
- **财务健康**:上市企业财报结构化(CSX),但柬埔寨供应商绝大多数是私企、财务不公开 → 任何自动路径都难,**主要靠用户上传补全**。
- **司法舆情**:柬埔寨无公开结构化司法库(法院判决 / 失信名单不开放),能拿的只有新闻媒体(非结构化)且媒体站反爬。字段(`negative_news_level`/`litigation_count` 等)**只能靠 LLM 从文章归纳,爬虫单独做不到** —— 四维度中对爬虫最不利。

## 五、未选用方案

| 方案 | 不选理由 |
|---|---|
| 自建爬虫(直连 MOC/媒体站) | PoC 实测三站全 403;非结构化源(司法)爬虫无法出字段;直连官方站合规风险;反爬对抗 + DOM 维护成本高 |
| 浏览器自动化(Playwright/Selenium) | 能绕部分反爬,但重、脆(CF 持续升级)、合规仍存疑、维护成本高,MVP 阶段否决 |
| 纯商业 API 为主路径 | OpenCorporates 等对柬埔寨覆盖有限且付费;列为 Δ8 增强,不作 MVP 主路径 |

## 六、后果

**优点**:
- 合规清晰(不直连官方站),柬埔寨可达(绕反爬)
- 与 Δ7 / ADR-0003 同栈,无新增技术依赖
- 非结构化维度(司法)有解(LLM 归纳)

**代价 / 风险**:
- 工商填充率偏低(1~3/7)——缓解:`search_depth=advanced`、商业 API 补全、用户上传
- 私企财务大面积缺失——缓解:用户上传财报;评分走缺失降级,不虚高(配合评分规则:缺失不应给中性高分,见后续评分规则评审)
- Tavily 配额 / qwen Token 成本——缓解:24h 缓存 + 限调用次数
- LLM 召回/抽取不稳定——缓解:Δ7 反幻觉 + 源追溯 + `raw_data` 留痕可溯源
